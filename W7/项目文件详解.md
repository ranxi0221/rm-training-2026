# 📁 project/armor_detector/ 全部文件详解

> 答辩时可以打开这个文件对着讲，每个文件是干什么的、在数据流里处于哪个位置。


## 一、编译和包声明（2个文件）

### `CMakeLists.txt` — 编译说明书

**干什么的：** 告诉 CMake 系统怎么把这个项目编译出来。

**核心内容：**
```
1. find_package  → 找到 OpenCV、rclcpp 这些依赖库
2. add_library   → 把 armor_detector.cpp + digit_classifier.cpp 编译成库
3. add_executable → 把 armor_detector_node.cpp 编译成可执行文件
4. target_link_libraries → 让可执行文件"链接"到刚才编译的库
5. install       → 告诉编译系统把文件装到哪里
```

**答辩讲法：** "CMakeLists.txt 定义了编译规则。算法部分编译成库，节点部分编译成可执行文件，两者链接在一起。这样算法可以被其他地方复用。"

### `package.xml` — ROS2 包身份证

**干什么的：** 告诉 ROS2 "这个包叫什么、依赖哪些东西"。

**核心内容：**
```xml
<name>armor_detector</name>       ← 包名，必须和文件夹名一致
<depend>rclcpp</depend>          ← 需要 ROS2 C++ 库
<depend>cv_bridge</depend>       ← 需要 ROS 和 OpenCV 之间的桥梁
<depend>OpenCV</depend>          ← 需要 OpenCV 库
```

**答辩讲法：** "package.xml 声明了包的基本信息和依赖。colcon build 会先检查这些依赖是否存在。"


## 二、核心代码（5个文件）

### `include/armor_detector/armor_detector.hpp` — 数据结构 + 检测器接口

**干什么的：** 定义了项目里用到的所有数据结构和检测器的对外接口。

**四个核心定义：**

| 结构体 | 内容 | 作用 |
|------|------|------|
| `LightBar` | 旋转矩形 + 颜色 + 置信度 | 描述一个检测到的灯条 |
| `ArmorPlate` | 四个角点 + 中心 + 颜色 + 数字 | 描述一个检测到的装甲板（含数字识别结果） |
| `DetectorParams` | 40+ 个参数 | 所有可调参数集中在一个结构体里 |
| `ArmorDetector` | 检测器类 | `detect()` 是入口，`drawDebug()` 画结果 |

**答辩讲法：** "头文件定义了 LightBar（灯条）和 ArmorPlate（装甲板）两个核心数据结构。ArmorPlate 里有一个 digit 字段是 W7 新增的，存储数字识别结果。DetectorParams 把所有参数收在一个结构体里，方便 YAML 文件传参。"

### `src/armor_detector.cpp` — 核心算法（1300+ 行）

**干什么的：** 整个项目最核心的文件。实现了从图像到装甲板结果的全部流程。

**函数调用链：**

```
detect()  ← 对外唯一入口
  │
  ├── preprocess()          → 降亮度 + CLAHE
  ├── cvtColor BGR→HSV
  ├── 如果亮模式 → 调整参数
  │
  ├── extractLightBars("red")   → 找红灯条
  │     ├── inRange（HSV 阈值）
  │     ├── BGR 纯度过滤 ← W7核心改进！
  │     ├── 形态学（开/闭运算）
  │     ├── findContours（找轮廓）
  │     └── 筛选（面积、宽高比）
  │
  ├── extractLightBars("blue")  → 找蓝灯条（同上）
  ├── 恢复原始参数
  ├── LED 验证（峰值/边缘比 > 2.5）
  │
  ├── matchArmorPlate()     → 配对
  │     ├── 按颜色分组
  │     ├── 同色全遍历配对
  │     ├── 几何约束（dx/dy）
  │     └── 尺寸相似性检查（高度/宽度比）
  │
  ├── 每色取面积最大
  ├── 数字识别
  ├── EMA 平滑
  └── drawDebug()           → 画框 + 文字标签
```

**答辩讲法：** "armor_detector.cpp 是核心算法文件。detect() 是总入口，调用了预处理、灯条提取、配对、数字识别、平滑、绘制这些子函数。W7 最重要的改动——BGR 纯度过滤——就在 extractLightBars 函数里，在 HSV 分离之后、形态学之前。"

### `src/armor_detector_node.cpp` — ROS2 节点

**干什么的：** 把算法（armor_detector.cpp）和 ROS2 世界连起来。

**三个角色：**
| 角色 | 做的事 |
|------|------|
| 订阅者 | 订阅 `/image_raw` 话题，接收相机图像 |
| 调用者 | 把图像传给 `detector_.detect()`，拿到装甲板列表 |
| 发布者 | 发布 `/armor_result`（文字结果）+ `/armor_debug_image`（画框图像）+ `/armor_mask`（黑白调试图） |

**答辩讲法：** "节点文件是一个胶水层，它本身不涉及任何检测逻辑。它的工作就是：收到图像→交给人算法→拿到结果→发布出去。这种分层设计让算法可以被任何 ROS2 节点复用。"

### `include/armor_detector/digit_classifier.hpp` — 数字识别头文件

**干什么的：** 声明数字识别分类器的接口。

```cpp
class DigitClassifier {
    bool loadModel(path);           // 加载 ONNX 模型
    string classify(bgr, corners);  // 输入原图+四角 → 输出 "1"/"3" 等
    Mat extractROI(bgr, corners);   // 从四角裁出 32×32 ROI
};
```

### `src/digit_classifier.cpp` — 数字识别实现（W7 新增）

**干什么的：** 实现数字识别的完整流程。

**extractROI 裁切逻辑：**
```
装甲板四角 → 外接矩形
  → 长方形（宽>高）：取中心 60% 宽 × 85% 高
  → 正方形（宽≈高）：取中心 35% 宽 × 85% 高（避开图标）
  → 灰度化 → 等比例缩放 → 32×32 黑底填充 → 归一化 [0,1]
```

**classify 推理流程：**
```
ROI blob → net_.setInput → net_.forward → 
9类置信度 → 取最大 → 返回类别字符串（"1"/"3"/"sentry"...）
```

**答辩讲法：** "digit_classifier 是 W7 新增的文件。extractROI 负责从装甲板四角裁出数字区域，根据装甲板形状自适应裁切宽度——长方形装甲板数字大、没图标，裁 60%；正方形有图标干扰，裁 35%。裁出来的区域缩放到 32×32，这是模型训练时用的尺寸。classify 函数用 OpenCV DNN 做推理，不需要额外的深度学习框架。"


## 三、配置和启动（2个文件）

### `config/armor_params.yaml` — 参数集中营

**干什么的：** 30+ 个参数收在一个文件里，改完重启就生效。

**关键参数速览：**
```yaml
# 蓝色灯条
blue_h_low: 100, blue_h_high: 125   # 色相范围（只取蓝紫色）
blue_s_low: 40                        # 饱和度下限（太低会抓白墙）
blue_v_low: 150                       # 亮度下限（太低会抓暗噪）

# 灯条筛选
light_min_area: 20                    # 最小面积（太小=噪点）
light_max_ratio: 0.6                  # 宽高比上限（太胖=反光）

# 配对
pair_max_dy: 400                      # Y偏差（太大=不在同一水平线）
pair_min_dx: 5, pair_max_dx: 2500    # X间距范围

# 平滑
smooth_alpha: 0.4                     # EMA系数（越大越防抖，越小越跟手）
```

**答辩讲法：** "所有参数在 YAML 里集中管理。这样有几个好处：第一，调参不需要重编译；第二，运行中可以实时改——ros2 param set 就能生效；第三，方便对比不同参数的效果。这个文件是调试过程中改得最频繁的文件。"

### `launch/armor.launch.py` — 一键启动脚本

**干什么的：** 一条命令启动所有需要的节点。

**核心逻辑：**
```python
input_source = 'bag'  # 默认 bag 模式，不需要相机驱动

if input_source == 'camera' → 启动 mindvision_camera 节点
if input_source == 'bag'    → 播放指定 bag 文件
检测节点 → 无条件启动（传入 YAML 参数 + 模型路径）
```

**答辩讲法：** "launch 文件的核心设计是 input_source 参数。默认 bag 模式意味着只编译 armor_detector 就能跑，不需要相机驱动。有相机的时候加 input_source:=camera 就行。"

## 七、数据流总图

```
ros2 launch armor.launch.py
  ├── 读 armor_params.yaml
  ├── [可选] 启动 mindvision_camera 或 播放 bag
  └── 启动 armor_detector_node
        │
        ├── 订阅 /image_raw
        ├── cv_bridge → cv::Mat
        ├── cv::flip（画面翻转）
        │
        ├── detector_.detect(bgr)
        │     ├── preprocess()
        │     ├── extractLightBars() × 2（红+蓝）
        │     │     ├── HSV inRange
        │     │     ├── BGR 纯度过滤 ← W7
        │     │     ├── 形态学
        │     │     └── 筛选灯条
        │     ├── LED 验证
        │     ├── matchArmorPlate()
        │     │     ├── 按颜色分组
        │     │     ├── 全遍历配对
        │     │     └── 尺寸相似性
        │     ├── 每色取最大
        │     ├── digit_classifier_.classify() ← W7
        │     ├── EMA 平滑
        │     └── drawDebug()
        │
        ├── 发布 /armor_result
        ├── 发布 /armor_debug_image
        └── 发布 /armor_mask


启动命令：
ros2 launch armor_detector armor.launch.py \
    input_source:=camera \
    target_color:=both \
    digit_model_path:=models/tiny_resnet.onnx

编译命令：
cd ~/ros2_ws
colcon build --packages-select armor_detector
source install/setup.bash
```
